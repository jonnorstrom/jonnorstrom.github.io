What are Ruby Classes?

Ruby classes are the gateway into getting your objects to do whatever you want them to do. You already know what classes are! Classes are Arrays, Strings, Hashes, etc. Let's say I have a class called 'Netflix', class Netflix would be how you create this class. Once I've created this class I can create my array of Netflix shows and movies that I'd like to watch, list = ["House of Cards", "Love", "Friends", "Narcos"]. Great! Now my class is created and it is an array of my shows I'd like to watch on Netflix. Next I just need to create two instance methods, named "Add" and "Remove".

A quick note about instance methods: instance methods are #methods that can specifically be used on my netflix class. For instance I would write netflix.add("Jessica Jones") and then netflix.remove("Friends") - because of course I've already seen all of Friends, a few times - and then my new output would look like this: list = ["House of Cards", "Love", Narcos", "Jessica Jones"].

So I've created my instance methods. But let's say I want to display my list nicely? Well to do that, I'll need to assign my 'list' to be an instance variable. This is a variable which can be accessed anywhere in my class - as long as this variable was created in my class space or in my instance method space. So now list = [...] becomes @list = [...] Now that my 'list' is an instance variable, I can create a new instance method that would go through my array and print all it's items out in a nice little list! Basically, an instance variable is a value that only exists in the given context. So @list in my Netflix class will mean something entirely different than a @list variable in my Grocery class. That list would have food and things on it! I do not want those things on my Netflix 'to-watch' list!
