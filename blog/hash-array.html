Hashes and Arrays

Hashes and arrays can get a bit confusing to any newcomers in the game such as myself. But I've been doing some reading and the difference is becoming tangible. Lets start out by saying that they have their fair share of similarites.

  • They are both indexes - this meaning they are both just lists (big or small). These lists can contain any amount of numbers, strings, objects, etc.
  • Their indexed items both, technically, can be called using a key (the keys are different, however, and I'll get back to this!)
  • Both allow for some code or method iteration

Now I'll spend some time explaining how they set each other apart.


Hashes vs. Arrays

The main difference is that hashes are stored pairs of information whereas arrays are just lists of information. Hashes are typically quicker when it comes to searching for a specific piece of information than arrays. While they both can have their elements specifically called by a key, the key in which you use is different for the two. For a hash the key is a word that you have chosen, and for arrays the key is just the index number of the item.
